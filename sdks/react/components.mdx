## PricingTable

`PricingTable` reads `GET /api/v1/plans` with `BillingProvider.publishableKey`. It does not fetch current-plan or customer data.

When no custom `onSelect` is supplied, its CTA calls `BillingProvider.createCheckout` with only:

```ts
{ planCode, returnUrl }
```

```tsx
<PricingTable
  currentPlanCode="free"
  highlightPlan="pro"
  returnUrl="https://wrrk.ai/settings/billing"
  onStripeClientSecret={setClientSecret}
/>
```

Pass `plans` to render caller-supplied catalog data without fetching it.

## CheckoutButton

```tsx
<CheckoutButton
  planCode="pro"
  returnUrl="https://wrrk.ai/settings/billing"
  onStripeClientSecret={setClientSecret}
  onComplete={() => refreshBillingStatus()}
  onScheduled={() => refreshBillingStatus()}
/>
```

The component invokes the merchant callback. It never sends a customer ID or calls Nozle directly. A missing `createCheckout` callback produces a clear error through `onError`.

## UpgradeButton and UpgradeModal

Both components forward `planCode` and `returnUrl` to the same merchant callback. `UpgradeModal` accepts an optional caller-supplied `preview`; it does not fetch a customer preview from the browser.

```tsx
<UpgradeButton
  planCode="max"
  returnUrl={window.location.href}
  preview={{ credit: 10, debit: 50, net: 40, nextBillingDate: '2026-08-01' }}
/>
```

## Checkout

`Checkout` renders a Stripe client secret returned by the merchant backend. Stripe webhook processing remains authoritative; a redirect or embedded completion callback alone does not activate a paid plan.

## Presentational components

`FeatureGate`, `PlanGate`, `UsageGate`, `UsageMeter`, `UsageDashboard`, `UsageAlert`, `PlanBadge`, and `PaymentMethodDisplay` consume caller-supplied state. Retrieve customer billing data from your authenticated backend before rendering them.
