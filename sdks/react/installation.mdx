## Configure browser and server authentication

Keep your Nozle secret key server-side only. A publishable key powers only the public plan catalog. Every customer-bound billing read or mutation requires a short-lived, scoped customer session.

For Vite:

```bash title=".env"
VITE_NOZLE_PUBLISHABLE_KEY=pk_nozle_...
VITE_NOZLE_API_URL=https://api.nozle.app
VITE_STRIPE_PUBLISHABLE_KEY=pk_live_...
```

For Next.js:

```bash title=".env.local"
NEXT_PUBLIC_NOZLE_PUBLISHABLE_KEY=pk_nozle_...
NEXT_PUBLIC_NOZLE_API_URL=https://api.nozle.app
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
```

`VITE_STRIPE_PUBLISHABLE_KEY` / `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` is only needed if your app mounts embedded Stripe checkout. If you only redirect to a hosted checkout URL, you do not need to mount Stripe Elements in your app.

Mint the least-privilege session in your authenticated backend route:

```ts title="app/api/nozle-session/route.ts"
const session = await nozle.customerSessions.create({
  customerId: currentCustomerId,
  expiresInSeconds: 900,
  scopes: [
    'credits:read',
    'billing:read',
    'entitlements:read',
    'checkout:create',
    'subscriptions:write',
    'topups:create',
  ],
});

return Response.json(session);
```
</Step>

## Wrap your app with BillingProvider

`BillingProvider` initializes the Nozle client and makes billing data available to all child components and hooks.

```tsx title="src/App.tsx"
import { BillingProvider } from '@nozle-js/react';

function App({ children, currentCustomerId, customerSessionToken }) {
  return (
    <BillingProvider
      publishableKey={import.meta.env.VITE_NOZLE_PUBLISHABLE_KEY}
      customerSessionToken={customerSessionToken}
      baseUrl={import.meta.env.VITE_NOZLE_API_URL ?? 'https://api.nozle.app'}
      customerId={currentCustomerId}
      workspaceId="ws_abc123"
    >
      {children}
    </BillingProvider>
  );
}
```

In Next.js, use `process.env.NEXT_PUBLIC_NOZLE_PUBLISHABLE_KEY` and `process.env.NEXT_PUBLIC_NOZLE_API_URL` instead of `import.meta.env.*`.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `customerSessionToken` | `string` | -- | Short-lived `csess_...` token for customer-bound reads and mutations. |
| `publishableKey` | `string` | -- | Client-safe key used only for the public plan catalog. |
| `apiKey` | `string` | -- | Deprecated alias: `pk_...` remains catalog-only and `csess_...` remains customer-only. Secret keys are rejected. |
| `customerId` | `string` | `''` | Your external customer ID. Required by customer-scoped hooks and billing components. |
| `baseUrl` | `string` | `'https://api.nozle.app'` | API endpoint |
| `workspaceId` | `string` | `''` | Your workspace ID |
| `children` | `ReactNode` | **required** | Your app content |

Provide `publishableKey` when rendering catalog components and `customerSessionToken` when rendering customer operations. The provider routes catalog requests to the publishable key and customer requests to the session; it never falls back from a missing session to the publishable key.

Customer credit and billing access is HTTP-only. Customer sessions cannot mint Centrifugo tokens and carry no realtime customer-credit claims.

<Warning>
Never expose your `sk_nozle_...` secret key in browser code. A publishable key is catalog-only. Use an explicitly scoped customer session for checkout, billing, entitlements, plan changes, cancellation, top-ups, and credit reads.
</Warning>
</Step>

## Configure Tailwind CSS

Nozle components ship with a Tailwind preset that provides consistent theming out of the box. Add it to your Tailwind config:

```js title="tailwind.config.js"
import { nozlePreset } from '@nozle-js/react/styles/preset';

export default {
  presets: [nozlePreset],
};
```

The preset registers the following CSS custom properties that you can override to match your brand:

| Variable | Purpose |
|----------|---------|
| `--nozle-primary` | Primary accent color |
| `--nozle-primary-foreground` | Text color on primary backgrounds |
| `--nozle-background` | Page background |
| `--nozle-foreground` | Default text color |
| `--nozle-card` | Card background color |
| `--nozle-border` | Border color |
| `--nozle-muted` | Muted background (toggles, disabled states) |
| `--nozle-muted-foreground` | Muted text color |
| `--nozle-destructive` | Error / danger actions |
| `--nozle-success` | Success indicators |
| `--nozle-warning` | Warning indicators |
| `--nozle-radius` | Border radius for components |

Some components also accept component-specific CSS variables for finer control. For example, `PricingTable` supports `--nozle-pricing-bg`, `--nozle-pricing-card-bg`, `--nozle-pricing-highlight`, `--nozle-pricing-border`, and `--nozle-pricing-radius` -- all falling back to their global equivalents.
