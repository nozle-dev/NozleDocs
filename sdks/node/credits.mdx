## Read balances and history

```ts
const balance = await nozle.credits.getBalance('customer_123', 'ai_credits');
console.log(balance.available); // exact decimal string

const allBalances = await nozle.credits.listBalances('customer_123');

const firstPage = await nozle.credits.listOperations('customer_123', {
  creditSystemCode: 'ai_credits',
  limit: 25,
});

const nextPage = firstPage.next_cursor
  ? await nozle.credits.listOperations('customer_123', {
      creditSystemCode: 'ai_credits',
      limit: 25,
      cursor: firstPage.next_cursor,
    })
  : null;
```

## Mint a React read session

```ts
const session = await nozle.customerSessions.create({
  customerId: 'customer_123',
  expiresInSeconds: 900,
});
```

Call this only from an authenticated backend route. Send `session.token` to `BillingProvider`; never send your `sk_` key to the browser.

## Manage Entities

Entity mutations require a secret key and a caller-supplied idempotency key:

```ts
await nozle.entities.upsert(
  'workspace_123',
  'user_42',
  { name: 'Asha', status: 'active', metadata: { role: 'agent' } },
  { idempotencyKey: 'entity-user-42-v1' },
);

const page = await nozle.entities.list('workspace_123', {
  status: 'active',
  limit: 50,
});

await nozle.entities.suspend('workspace_123', 'user_42', {
  idempotencyKey: 'suspend-user-42',
});
```

`suspend` and `activate` first read the Entity and preserve its current name and metadata. Retry an uncertain mutation with the same idempotency key.

Entity reads include nullable `seat_sync` state when lifecycle seat billing applies. Inspect `operation_type`, publication `status`, `last_reconciled_at`, `reconciliation_status`, and expected/observed counts before treating the billing projection as synchronized. A `null` value means no seat-event synchronization projection or history exists for that Entity yet; it does not prove that no seat metric is configured.

## Read and transfer Entity credits

```ts
const balance = await nozle.credits.getEntityBalance(
  'workspace_123',
  'user_42',
  'ai_credits',
);

console.log(balance.entity_available);
console.log(balance.shared_available);
console.log(balance.effective_available);

const history = await nozle.credits.listEntityOperations(
  'workspace_123',
  'user_42',
  { creditSystemCode: 'ai_credits', limit: 25 },
);
```

Allocation/deallocation is backend-only, exact-decimal, and subject to the Engine's default-off exact transfer gate:

```ts
await nozle.credits.allocate(
  'workspace_123',
  'user_42',
  { creditSystemCode: 'ai_credits', amount: '100.000000000001' },
  { idempotencyKey: 'allocate-user-42-100' },
);
```

## Check and consume

```ts
const check = await nozle.usage.check({
  customerId: 'customer_123',
  billableMetricCode: 'agent_execution',
  creditSystemCode: 'ai_credits',
});

console.log(check.projected_remaining); // exact decimal string
console.log(check.projected_deductions); // ordered, no mutation

const result = await nozle.usage.track(
  {
    customerId: 'customer_123',
    entityId: 'user_42', // optional; enables Entity attribution and pool policy
    billableMetricCode: 'agent_execution',
    creditSystemCode: 'ai_credits',
    timestamp: new Date().toISOString(),
  },
  { idempotencyKey: 'execution_0183f' },
);
```

`usage.check` is advisory. Its projected source plan is useful for shadow
fixtures, but another request can spend credits before a later mutation. Only
`usage.track` atomically reserves and deducts value.
