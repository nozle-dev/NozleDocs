---
title: Real-Time Updates
description: WebSocket for live entitlement changes
---

Nozle supports WebSocket delivery for server-authorized real-time entitlement updates. The minimum customer-session contract is HTTP-only: neither publishable keys nor customer sessions can mint a realtime token.

**How it works:**
1. BillingProvider connects to WebSocket on mount
2. Subscribes to channel: workspace:\{workspaceId\}:entitlements
3. When entitlements change (plan upgrade, usage limit hit, feature toggled), Nozle publishes to the channel
4. React hooks (useCan, usePlan) update automatically — no polling

Do not send `pk_`, `csess_`, or `sk_` credentials to a WebSocket endpoint. Until your backend implements a separate server-authorized realtime-token flow, omit `centrifugoUrl` and use the HTTP hooks documented in this guide.

**Connection states:**
The BillingState tracks connection:
```typescript
const { connectionState } = useBillingContext();
// 'connecting' | 'connected' | 'disconnected'
```

**When updates are pushed:**
- Customer upgrades/downgrades plan
- Usage counter crosses a limit threshold
- Admin toggles a feature
- Subscription is cancelled or expired

**Without WebSocket:**
If `centrifugoUrl` is not provided, the React SDK uses REST-only mode. Hooks fetch data on mount but do not receive live updates. This is the supported mode for customer sessions.
