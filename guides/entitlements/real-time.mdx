---
title: Real-Time Updates
description: WebSocket for live entitlement changes
---

Nozle supports WebSocket delivery for server-authorized real-time entitlement updates. Browser publishable keys cannot mint a realtime token.

**How it works:**
1. BillingProvider connects to WebSocket on mount
2. Subscribes to channel: workspace:\{workspaceId\}:entitlements
3. When entitlements change (plan upgrade, usage limit hit, feature toggled), Nozle publishes to the channel
4. React hooks (useCan, usePlan) update automatically — no polling

Do not send `pk_` or `sk_` credentials to a WebSocket endpoint. Until your backend implements a separate server-authorized realtime-token flow, omit realtime configuration and use authenticated backend reads.

**Connection states:**
The BillingState tracks connection:
```typescript
const { connectionState } = useBillingContext();
// 'connecting' | 'connected' | 'disconnected'
```

**When updates are pushed:**
- Customer upgrades/downgrades plan
- Usage counter crosses a limit threshold
- Admin toggles a feature
- Subscription is cancelled or expired

**Without WebSocket:**
The React SDK does not fetch customer entitlement state. Supply caller-owned state to presentational gates, or add a separately authenticated merchant realtime channel.
