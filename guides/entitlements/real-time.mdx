---
title: Realtime security boundary
description: Why customer billing remains HTTP-only and how trusted realtime token creation is protected
---

Customer sessions cannot create Centrifugo tokens or subscribe to customer-credit channels. The React SDK uses authenticated HTTP for customer billing, entitlement, and product-credit reads.

This boundary prevents a browser session from expanding its exact customer scopes into a broader workspace channel. A publishable key is also rejected by the realtime-token route.

Refresh or remount the relevant browser query after checkout, a plan mutation, cancellation, or top-up completion. The current React provider deliberately supplies no customer-session Centrifugo token.

If a trusted server integration needs a realtime token, it must call `GET /api/v1/auth/centrifugo-token` with a secret key and keep that key out of the browser. Publishable keys and customer sessions receive `403 Forbidden`.

Do not proxy the secret-key token route directly to an untrusted browser. Define an application-specific, least-privilege channel contract before exposing any realtime capability.
