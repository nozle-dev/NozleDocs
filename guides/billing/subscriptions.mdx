**View current subscription:**

React: CurrentPlan component or GET /api/v1/subscriptions/current
```tsx
<CurrentPlan customerId="cust_123" onChangePlan={() => router.push('/plans')} />
```

**Upgrade/Downgrade:**

The UpgradeButton component handles the full flow:
1. Shows proration preview (calls POST /api/v1/subscriptions/preview)
2. Displays credit (unused time on current plan), debit (new plan cost), net amount, next billing date
3. On confirm, calls `POST /api/v1/checkout`
4. If payment is due, the current plan stays active while Stripe Checkout is shown
5. The upgrade applies after the Stripe success webhook; a fully credit-funded change completes immediately
6. A lower annualized plan, including some discounted monthly-to-annual switches, is scheduled at Lago's next billing boundary

```tsx
<UpgradeButton
  targetPlanId="scale"
  label="Upgrade to Scale"
  onStripeClientSecret={(secret) => setClientSecret(secret)}
  onUpgraded={() => toast('Upgraded with prepaid credits')}
/>
```

Or via SDK:
```typescript
// Preview proration first
const preview = await fetch('/api/v1/subscriptions/preview', {
  method: 'POST',
  headers: { Authorization: 'Bearer pk_...', 'Content-Type': 'application/json' },
  body: JSON.stringify({ target_plan_id: 'scale' })
});

// Start payment-first checkout
const checkout = await fetch('/api/v1/checkout', {
  method: 'POST',
  headers: { Authorization: 'Bearer pk_...', 'Content-Type': 'application/json' },
  body: JSON.stringify({ customer_id: 'cust_123', plan_code: 'scale' })
});
```

**Cancel subscription:**

```tsx
<CancelSubscriptionButton subscriptionId="sub_123" onCancelled={() => router.push('/')} />
```

Two-step flow: confirm → select reason (Too expensive, Missing features, Switching to competitor, Not using it enough, Other).
Calls `DELETE /api/v1/subscriptions/{id}` with the selected reason.
