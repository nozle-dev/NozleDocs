Nozle uses Stripe's embedded checkout for payment collection. The flow:

1. Customer selects a plan (PricingTable or CheckoutButton component)
2. React SDK calls `POST /api/v1/checkout` with `{ customer_id, plan_code, return_url }`
3. Nozle keeps the current plan active and computes the exact invoice/proration
4. If money remains due, Nozle creates a Stripe embedded Checkout Session
5. Returns `{ type: "stripe", client_secret }`
6. The Checkout component renders Stripe Embedded Checkout using the `cs_...` secret
7. Customer enters payment details
8. Stripe sends `payment_intent.succeeded` or `payment_intent.payment_failed` to Nozle
9. Nozle marks the invoice paid/failed and applies the paid plan change only after success

<Info>
Plan checkout is payment-gated: a paid plan change is not applied until Stripe confirms payment through the webhook. If prepaid credits reduce the exact amount due to zero, Nozle applies the change transactionally and returns `type: "completed"` without creating a Stripe session.
</Info>

## Required Stripe webhook setup

When you connect a Stripe account, Nozle registers a Stripe webhook endpoint for that account and stores Stripe's webhook signing secret. Your Stripe account must allow webhook endpoint creation.

The webhook target is derived from your public Nozle API URL:

```text
{NOZLE_PUBLIC_API_URL}/webhooks/stripe/{organization_id}?code={stripe_provider_code}
```

Stripe events used by checkout:

- `payment_intent.succeeded`
- `payment_intent.payment_failed`
- `payment_intent.canceled`

Without this webhook, Stripe may collect payment but Nozle will not reliably update invoice or subscription state.

## API shape

Plan checkout:

```json
{
  "customer_id": "customer_123",
  "plan_code": "growth",
  "return_url": "https://app.example.com/settings/billing"
}
```

Free plans and paid upgrades fully funded by prepaid credits return a completed response instead. Downgrades and other cycle-end transitions return a scheduled response from the same `/checkout` endpoint; browser sessions never call `/billing/upgrade` or `/subscriptions/change`.

## Response

```json
{
  "type": "stripe",
  "client_secret": "cs_...",
  "clientSecret": "cs_...",
  "invoice_id": "invoice_uuid",
  "amount_cents": 19900,
  "currency": "USD"
}
```

The browser calls your authenticated merchant backend, which derives the Nozle customer and starts checkout with a restricted `sk_`. A Nozle publishable key is valid only for the plan catalog. If you mount embedded Stripe checkout yourself, your app also needs the Stripe publishable key for the same Stripe account connected in Nozle.

Credit-funded completion response:

```json
{
  "type": "completed",
  "status": "succeeded",
  "payment_source": "credits",
  "plan_code": "growth",
  "amount_cents": 0,
  "currency": "USD"
}
```

Scheduled transition response:

```json
{
  "type": "scheduled",
  "status": "pending",
  "subscription_id": "subscription_uuid",
  "plan_code": "starter"
}
```

## React implementation

### Option A -- CheckoutButton (simplest)

```tsx
<CheckoutButton planId="growth" label="Subscribe to Growth" />
```

### Option B -- Custom flow with useCheckout

```tsx
const { fetchClientSecret, isLoading } = useCheckout();

const handleCheckout = async () => {
  const clientSecret = await fetchClientSecret('growth', '/success');
  // Pass the cs_... secret to <Checkout>
};
```

### Option C -- Embedded Stripe Checkout

```tsx
<Checkout
  clientSecret={secret}
  publishableKey={import.meta.env.VITE_STRIPE_PUBLISHABLE_KEY}
  onComplete={() => router.refresh()}
/>
```

## Server-side plan changes

For B2B scenarios where payment is already handled outside embedded checkout:

```typescript
const result = await nozle.subscribe('customer_123', 'growth');
// { subscription_id: 'sub_...', status: 'active' }
```

`subscribe()` requires a secret key (`sk_`). Embedded checkout should be used when Nozle needs to collect payment before activating a paid plan.
