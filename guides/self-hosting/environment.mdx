---
title: Environment Variables
description: Configuration for self-hosted deployments
---

When self-hosting, you'll need to configure environment variables for the API, frontend, and worker processes. Nozle Cloud handles all of this automatically.

## Core Configuration

Key variables include:

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `REDIS_URL` | Redis connection string |
| `API_URL` | Self-referencing API URL |
| `SMTP_ADDRESS` | SMTP server for invoice emails |
| `SMTP_PORT` | SMTP port (typically 587) |
| `SMTP_FROM_EMAIL` | From address for transactional emails |

## Nozle API Keys

Regardless of deployment model, you'll need Nozle API keys to use the SDKs:

| Key Type | Prefix | Use |
|----------|--------|-----|
| Publishable | `pk_` | Browser plan catalog only |
| Secret | `sk_` | Server-side (Node/Python SDKs) |

## Security boundary

| Variable | Description |
|----------|-------------|
| `NOZLE_INTERNAL_SECRET` | Dedicated Core–Engine service credential of at least 32 bytes. Never reuse an organization API key. |
| `NOZLE_CUSTOMER_SESSION_SECRET` | Separate high-entropy HMAC secret of at least 32 bytes, used to sign scoped, short-lived customer sessions. |
| `NOZLE_METRICS_SECRET` | Separate secret required by Core and Engine operational metrics endpoints. |
| `NOZLE_CORE_WEBHOOK_PUBLIC_KEY` | Base64-encoded PEM RSA public key used by Engine to verify Core webhook signatures. |
| `NOZLE_CORE_WEBHOOK_ISSUER` | Exact expected Core webhook issuer. |
| `CORS_ORIGIN` | Transitional single-origin setting retained for old Engine replicas during a rolling deployment. |
| `CORS_ALLOWED_ORIGINS` | Comma-separated browser-origin allowlist, for example `https://app.example.com,https://portal.example.com`. |
| `CHECKOUT_RETURN_URL` | Independent absolute URL used when checkout requests omit a return URL. Do not reuse this as the CORS allowlist. |
| `CHECKOUT_RETURN_URL_ALLOWED_ORIGINS` | Exact HTTPS origin allowlist for caller-supplied checkout return URLs. |
| `NOZLE_CREDIT_USAGE_TRACK_TARGETS` | Preferred comma-separated exact mutation targets in `organization_uuid:metric_code` or `organization_uuid:metric_code:credit_system_code` form. Any configured exact target list takes precedence over the legacy organization allowlist. Leave empty by default. |
| `NOZLE_CREDIT_USAGE_TRACK_ORGANIZATION_IDS` | Legacy comma-separated organization-wide mutation allowlist. Leave empty when using exact targets. |
| `NOZLE_ENTITY_SEAT_TARGETS` | Comma-separated exact `organization_uuid:metric_code` targets for Entity lifecycle seat publication and active-count reconciliation. One metric per organization. Leave empty by default and deploy Engine migration 011 before enabling. |
| `NOZLE_ENTITY_CREDIT_TRANSFER_TARGETS` | Comma-separated exact `organization_uuid:credit_system_code` external transfer targets. Leave empty by default. |

Rotate `NOZLE_CUSTOMER_SESSION_SECRET` deliberately: replacing it immediately invalidates every outstanding customer session. Never expose it through frontend build variables.

During the environment rename, set all three CORS/checkout variables. New Engine versions prefer `CORS_ALLOWED_ORIGINS` and `CHECKOUT_RETURN_URL`; old versions continue reading `CORS_ORIGIN`. Remove `CORS_ORIGIN` only after all old Engine versions have been retired.

Get your keys from the Nozle dashboard at [app.nozle.app](https://app.nozle.app).
