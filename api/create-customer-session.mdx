---
title: Create Customer Session
api: "POST /customer-sessions"
description: Mint a short-lived customer-bound token for browser credit reads
---

**Auth:** secret key (`sk_nozle_...`) only. Call this endpoint from your backend.

```json
{
  "customer_id": "customer_123",
  "expires_in_seconds": 900
}
```

`expires_in_seconds` is optional, defaults to 900 seconds, and must be between 60 and 3600.

```json
{
  "customer_session": {
    "token": "csess_...",
    "customer_id": "customer_123",
    "expires_at": "2026-07-20T12:15:00Z",
    "scope": ["credits:read"]
  }
}
```

The customer must exist in the authenticated organization. The token is accepted by these bounded HTTP reads for that customer:

- customer Credit System list, single-system balance, and operation history;
- Entity Credit System list, single-system balance, and operation history for an Entity owned by that customer.

It cannot list the customer's Entity directory, mint a Centrifugo token, subscribe to customer-credit realtime channels, track usage, create purchases, modify billing, or read another customer.

Issue it after authenticating the user in your own application, keep it in memory where possible, and refresh it from your backend after expiry. Do not place a Nozle secret key in the browser or persist the session in long-lived local storage.

```ts
const session = await nozle.customerSessions.create({
  customerId: "customer_123",
  expiresInSeconds: 900,
});
```
