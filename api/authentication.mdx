## Base URLs

The Nozle API has two base URLs depending on the type of endpoint:

| Base URL | Endpoints | Description |
|----------|-----------|-------------|
| `https://api.nozle.app/api/v1` | Entitlements, Checkout, Margin, Cost Models, Credits, LLM Proxy | Nozle intelligence layer — features built on top of the billing engine |
| `https://core.nozle.app/api/v1` | Customers, Events, Plans, Subscriptions, Invoices, Wallets, and all other billing resources | Full billing engine API |

<Info>
Each endpoint page shows the correct base URL in the code examples on the right. You don't need to memorize which URL to use — just copy the snippet.
</Info>

## API Keys

**Three credential types:**

| Key Type | Prefix | Use | Example |
|----------|--------|-----|---------|
| Publishable | `pk_nozle_` | Public plan catalog only | `pk_nozle_abc123...` |
| Secret | `sk_nozle_` | Server-side only (Node/Python SDKs, cURL) | `sk_nozle_xyz789...` |
| Customer session | `csess_` | Short-lived browser operations for one customer | `csess_eyJ...` |

All authenticated endpoints require a Bearer token in the Authorization header:

```
Authorization: Bearer sk_nozle_your_secret_key
```

## Key Scoping

- **Publishable keys** (`pk_nozle_...`) may call only `GET /api/v1/plans`. Every customer read and mutation returns HTTP 403.
- **Secret keys** (`sk_nozle_...`) are server-side credentials for configuration, usage mutation, customer-session creation, and privileged reads.
- **Customer sessions** (`csess_...`) are bound to one organization and exactly one external customer ID, expire after at most one hour, and carry an explicit subset of `credits:read`, `billing:read`, `entitlements:read`, `checkout:create`, `subscriptions:write`, and `topups:create`. Every customer ID in the path, query, or body must match the session subject.

Create customer sessions from your backend with `POST /api/v1/customer-sessions`. Use the publishable key only for catalog requests and the customer session for billing status, invoices, checkout, cancellation, top-ups, entitlements, and credit reads. Customer sessions cannot call `/subscribe`, mutate usage, or mint Centrifugo tokens.

## Public Endpoints

These do not require an API key:

- `POST /api/v1/auth/send-otp`
- `POST /api/v1/auth/verify-otp`
- `POST /api/v1/auth/signup`

## Getting Your Keys

Sign in at [app.nozle.app](https://app.nozle.app) → Settings → API Keys.

<Warning>
Never expose secret keys in client-side code. Use publishable keys only for the public plan catalog and a short-lived customer session for every customer-specific browser operation.
</Warning>
