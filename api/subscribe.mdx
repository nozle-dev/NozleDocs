---
title: "Create Subscription"
api: "POST /subscribe"
---

Create a subscription directly from a trusted backend without going through the browser checkout flow. This endpoint never accepts a publishable key or customer session, including during the staged compatibility window. Browser paid-plan changes must use `POST /checkout`.

**Auth:** `sk_` (secret key) only.

The secret key requires `subscription:write`, and the customer must belong to the authenticated organization.

<ParamField body="customer_id" type="string" required>
  External customer ID. The customer must already exist.
</ParamField>

<ParamField body="plan_code" type="string" required>
  The plan code to subscribe the customer to (e.g. `pro`).
</ParamField>

### Response

<ResponseField name="subscription_id" type="string" required>
  The newly created subscription's ID.
</ResponseField>

<ResponseField name="status" type="string" required>
  Subscription status (e.g. `active`).
</ResponseField>
